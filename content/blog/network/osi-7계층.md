## OSI 7계층

- __`OSI 7계층(Open System Interconnection 7 Layer)`은 국제 표준화 단체인 ISO에서 제시한 표준화된 네트워크 구조 모델이다.__  

    |                  |                      |
    |------------------|----------------------|
    | 응용 계층        | 7 Application Layer  |
    | 표현 계층        | 6 Presentation Layer |
    | 세션 계층        | 5 Session Layer      |
    | 전송 계층        | 4 Transport Layer    |
    | 네트워크 계층    | 3 Network Layer      |
    | 데이터 링크 계층 | 2 Data Link Layer    |
    | 물리 계층        | 1 Physical Layer     |
</br>
- 통신기능을 7계의 계층으로 분할하여 각 계층마다 다른 계층과는 무관하게 자신의 독립적인 기능을 지원하도록 구성했다.  
</br>

- 계층을 나눔으로써 통신이 진행되는 과정을 단계별로 파악하고 보완할 수 있다. 또한, __한 모듈에 대한 변경이 전체 모듈에 미치는 영향을 최소화할 수 있다.__  
</br>

- 일반 사용자는 응용 계층을 통해 데이터의 송수신을 요청하며, 이 요청은 하위 계층으로 순차적으로 전달되어 맨 아래에 있는 물리 계층을 통해 상대 호스트에 전송된다. 요청이 각 계층으로 하달되는 과정에서 __계층별로 담당하는 기능을 수행해__ 데이터를 안전하게 전달한다.  

## 물리 계층

- __`물리 계층(Physical Layer)`은 전기적, 기계적, 기능적 특성을 이용해 통신 케이블로 데이터를 전송한다.__  

- 데이터 전송 단위로 `비트(bit)`를 사용한다.  

- 케이블, 리피터, 허브 등의 장비가 이에 해당한다.  

## 데이터 링크 계층

- __`데이터 링크 계층(Data Link Layer)`은 점대점의 신뢰성있는 전송을 담당한다.__  

- 신뢰성있는 전송을 위해 에러 검출 및 정정, 흐름 제어를 수행한다.  
</br>

    __1. `에러 검출 및 정정(Error Correction)`__  
    (1) FEC(Forward Error Control): 수신측에서 오류를 스스로 검출하고 정정한다.  

    (2) BEC(Backward Error Control): 송신측에서 수신측에게 오류 사실을 알리고 재전송을 받아 복원한다.

    __2. `흐름 제어(Flow Control)`__  
    (1) Stop and Wait: 송신측에서 프레임을 전송한 후 확인 응답을 받을 때 까지 대기한다.  

    (2) Sliding Window: 송신측에서 수신측의 수신 가능한 범위 내에서 여러 개의 프레임을 전송한다.  
</br>

- 물리적인 주소 값인 `MAC Address`를 담당한다.

- 데이터 전송 단위로 `프레임(frame)`을 사용한다.

- 대표적으로 `이더넷`이 있다.

## 네트워크 계층

- __`네트워크 계층(Network Layer)`은 패킷을 출발지로부터 여러 네트워크를 통해 목적지까지 전달한다.__  
</br>

    __1. `라우팅(Routing)`__  
    - __출발지에서 목적지까지 패킷의 경로를 결정한다.__  
    - 경로를 계산하는 알고리즘을 라우팅 알고리즘이라 한다.  

    __2. `포워딩(Forwarding)`__  
    - __패킷이 라우터의 입력 링크에 도달했을 때 그 패킷을 적절한 출력 링크로 이동시킨다.__  
</br>

- 논리적인 주소 값인 `IP Address`를 담당한다.

- 데이터 전송 단위로 `패킷`(packet)을 사용한다.  

- 대표적으로 `IP`(Internet Protocol)이 있다.  

## 전송 계층

- __`전송 계층(Transport Layer)`는 송신 프로세스와 수신 프로세스 간의 연결을 제공하여 프로세스 사이의 안전한 데이터 전송을 지원한다.__  

- 이를 위해 오류 검출과 흐름 제어를 수행한다.  

- 데이터 전송 단위로 `세그먼트`(segment) 혹은 `데이터그램`(datagram)을 사용한다.  

- 대표적으로 TCP와 UDP가 있다.  
</br>

    __1. `TCP(Transmission Control Protocol)`__  
    - __연결형 서비스이다.__  
    - __3-way handshake로 연결하고, 4-way handshake로 연결을 해제한다.__  
    - 흐름 제어(Flow Control)와 혼잡 제어(Congestion Control)를 제공한다.  
    - 데이터를 `세그먼트` 형태로 보낸다.  
    - UDP보다 느리지만 신뢰성을 보장한다.  
    - 전이중(전송이 양방향으로 이루어짐)으로 연결된다.  

    __2. `UDP(User Datagram Protocol)`__  
    - __비연결형 서비스이다.__  
    - 연결을 위한 논리적인 경로가 없다.
    - 데이터를 `데이터그램` 단위로 처리한다.  
    - 신뢰성이 낮으나 TCP보다 빠르다.  
    - 따라서 신뢰성보다 속도가 중요한 실시간 서비스에 사용된다.  

## 세션 계층

- __`세션 계층(Session Layer)`은 종단(End to End)간 어플리케이션의 세션을 구축하고 관리하며 제거한다.__  

## 표현 계층

- __`표현 계층(Presentation Layer)`은 전송되는 데이터의 의미에 관련되어 인코딩, 압축, 암호화를 담당한다.__

- EBCDIC으로 인코딩된 파일을 ASCII로 인코딩된 파일로 바꾸거나, 동영상과 같은 대용량 데이터를 압축하거나, 외부 침입자로부터 데이터를 안전하게 암호화할 수 있다.

## 응용 계층

- __`응용 계층(Application Layer)`은 사용자에게 통신을 위한 각종 서비스를 제공한다.__

- 사용자와 가장 가까이에 있는 계층이며, 어플리케이션을 통해 사용자와 응용 계층은 직접적으로 상호 작용한다.

## 마무리

1. `OSI 7계층`은 네트워크에서 통신이 일어나는 과정을 7단계로 나눈 것이다.
2. `물리 계층`(Physical Layer)은 통신 케이블로 데이터를 전송한다.
3. `데이터 링크 계층`(Data Linke Layer)은 MAC Address를 사용하며 에러 검출, 흐름 제어를 통해 신뢰성있는 프레임 전송을 수행한다.
4. `네트워크 계층`(Network Layer)은 IP Address를 사용하며 라우팅과 포워딩을 통해 패킷을 전달한다.
5. `전송 계층`(Transport Layer)은 TCP, UDP 프로토콜로 프로세스간의 안전한 데이터 전송을 지원한다.
6. `세션 계층`(Session Layer)은 종단간 어플리케이션의 통신 세션을 구축, 관리, 제거한다.
7. `표현 계층`(Presentation Layer)은 전송되는 데이터의 의미와 관련하여 인코딩, 압축, 암호화를 담당한다.
8. `응용 계층`(Application Layer)은 사용자와 가장 가까이에 있으며 사용자에게 통신을 위한 각종 서비스를 제공한다.
