## 인터페이스

인터페이스는 다음과 같이 생겼다.
```java
interface Bounceable {
    int PI = 3.14;
    void setBounce();
}
```

- __`인터페이스`(Interface)는 어떤 클래스가 구현(implement)해야 할 `행위`를 명시해 놓은 일종의 `추상 자료형`이다.__  
</br>

- 인터페이스에 존재할 수 있는 메소드에는 추상 메소드, 디폴트 메소드, static 메소드가 있다.  
</br>

- setBounce()처럼 __몸체가 비어있는 메소드를 가리켜 `추상 메소드`(Abstract method)라 한다.__  
</br>

- 다른 클래스가 인터페이스 내에 위치하는 추상 메소드들을 구현해야 하기 때문에 이들은 __별도의 선언이 없어도 `public`으로 간주한다.__  
</br>

- 인터페이스에도 변수를 선언할 수 있으며, 이 변수들은 다음의 특징을 가진다.
    1. 반드시 선언과 동시에 값으로 초기화를 해야 한다.
    2. 모든 변수는 __public, static, final__이 선언된 것으로 간주한다.
    3. __결론적으로 인터페이스 내부에서 선언된 변수는 `상수`이다.__

## 인터페이스의 구현

```java
class RedBall extends Ball implements Bounceable, Movable {
    ...
    @Override
    public void setBounce() {...}
}
```

- 클래스에서 구현할 인터페이스를 명시할 때 키워드 `implements`를 사용한다.  
</br>

- 한 클래스는 둘 이상의 인터페이스를 동시에 구현할 수 있다. `(다중 상속)`  
</br>

- 상속(extends)과 구현(implements)은 동시에 가능하다.  
</br>

- 인터페이스의 추상 메소드와 이를 구현하는 메소드 사이에 `오버라이딩` 관계가 성립한다. 따라서 어노테이션 @Override의 선언이 가능하다.

```java
interface Printable {
    void print(String doc);
}

interface ColorPrintalbe extends Printable {
    void printCMYK(String doc);
}

class Prn909Drv implements ColorPrintable {
    @Override
    public void print(String doc) { 
        System.out.println("From black&white ver");
        System.out.println(doc);
    }

    @Override
    public void printCMYK(String doc) {
        System.out.println("From CMYK ver");
        System.out.println(doc);
    }
}
```

- `인터페이스 간 상속`도 가능하다. 단, 이때 키워드 `extends`를 사용한다.  
</br>


## 추상 클래스

추상 클래스는 다음과 같이 생겼다.
```java
public abstract class Demo {
    abstract public int sum(int x, int y);
    public int product(int x, int y) { return x*y; }
}
```

- __`추상 클래스`(Abstract class)는 `하나 이상의 추상 메소드`를 갖는 클래스를 말한다.__  
</br>

- 다른 클래스들과 마찬가지로 __인스턴스 변수와 인스턴스 메소드를 가질 수 있지만,__ 추상 클래스가 하나 이상 있는 경우 추상 클래스라고 부른다.  
</br>

- __추상 클래스를 대상으로 인스턴스 생성이 불가능하며, 다른 클래스에 의해 추상 메소드가 구현되어야 한다.__  
</br>

- 추상 클래스는 선언부에 `abstract` 선언이 명시되어야 한다.  
</br>

- 추상 클래스는 인터페이스와 성격이 유사하다.


## 추상 클래스의 구현

- 클래스이므로 키워드 `extends`를 사용한다.

```java
public class Calculation extends Demo {
    public int sum(int x, int y) { 
        return x+y; 
    }
    public int product(int x, int y) {
        return x*y; 
    }
}
```

- __추상 클래스의 추상 메소드는 반드시 하위 클래스에서 재정의되어야 한다.__

## 마무리

인터페이스는
1. `클래스가 구현해야 할 행위`를 명시해 놓은 것이다.
2. 상수와 추상 메소드, 디폴트 메소드, static 메소드를 가질 수 있다.
3. `두 개 이상의 인터페이스`를 한 번에 구현할 수 있다. (다중 상속)
4. 구현하는 클래스에서 키워드 `implements`를 사용한다.

추상 클래스는
1. `하나 이상의 추상 메소드`를 갖는 클래스이다. 
2. 인스턴스 변수와 인스턴스 메소드를 가질 수 있다.
3. 클래스이기 때문에 `다중 상속이 불가능`하다.
4. 구현하는 클래스에서 키워드 `extends`를 사용한다.
