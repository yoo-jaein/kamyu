## 자바의 특징
1. 객체지향 언어
자바는 `객체지향 프로그래밍 언어`로 현재 활발히 사용되고 있는 프로그래밍 언어 중 하나이다. 특히 웹 어플리케이션 개발 언어로 가장 많이 사용되고 있다.  

2. 높은 호환성
`"write once, run anywhere"`라는 슬로건 하에, 한 번 구현한 코드를 플랫폼에 독립적으로 사용할 수 있도록 설계되었다. 즉, 운영체제의 종류에 관계없이 대부분의 시스템에서 실행 가능하다. 

## 자바 프로그램의 실행 원리
- 일반적인 프로그램은 윈도우 또는 리눅스와 같은 운영체제 위에서 실행이 된다. 즉, 하드웨어를 기반으로 운영체제가 동작하고, 운영체제 위에서 프로그램이 실행되는 구조이다.   

- 하지만 자바 프로그램은 이와 조금 다르다. __자바는 운영체제가 자바 가상머신을 실행시키고, `자바 가상머신`이 자바 프로그램을 실행시키는 구조로 동작한다.__

## 자바 가상머신(JVM, Java Virtual Machine)
- __자바 가상머신은 자바 프로그램을 실행하기 위한 가상의 기계를 소프트웨어로 구성한 것이다.__ 운영체제 위에서 동작하며 자바 바이트코드를 CPU나 OS에 관계없이 동일한 형태로 실행시킬 수 있다.   

- 일반적으로 컴퓨터 프로그램은 사람이 이해하기 쉬운 고급 프로그래밍 언어를 사용해 작성되고 컴파일 과정을 통해 특정 CPU에서 실행 가능한 기계어 코드로 변환된다. 하지만 자바 프로그램은 `자바 바이트코드`(bytecode)라는 특수한 형태로 컴파일 된 후, `자바 가상머신` 상에서 실행된다. 따라서 JVM이 설치 가능하다면 __어떠한 플랫폼에서도 실행 가능하다.__  

- 윈도우에는 윈도우 전용 JVM이 있고, 리눅스에는 리눅스 전용 JVM이 있다. 따라서 운영체제에 따라 적절한 가상머신 설치를 해야 한다.

## 자바 컴파일러(Java Compiler)
- __자바 컴파일러는 자바 가상머신이 이해할 수 있는 코드인 `자바 바이트코드`(bytecode)를 생성한다.__  

- 소스코드는 자바 컴파일러에 의해 컴파일되어 확장자가 `.class`인 클래스 파일에 바이트코드가 담기게 된다.  

- 컴파일러에 의해서 생성되는 명령어의 크기가 기본적으로 1바이트이기 때문에 붙여진 이름이다. 

## 마무리
1. 자바 프로그램은 플랫폼에 독립적이다.
2. 플랫폼이 독립적일 수 있는 이유는 `자바 가상머신`위에서 자바 프로그램이 동작하기 때문이다.
3. `자바 가상머신`에서 `자바 바이트코드`를 읽어서 자바 프로그램을 실행한다.
4. `자바 바이트코드`는 `자바 컴파일러`에 의해 소스코드가 컴파일되어 확장자가 `.class`인 클래스 파일에 담기게 된 것이다.
