## 객체 지향의 다형성

`다형성`(polymorphism)이라는 단어는 원래 `여러 개의 형태를 갖는다`라는 의미의 그리스어에서 유래했다. 객체 지향 프로그래밍에서 다형성의 개념이 적용된 것이 바로 `오버로딩`과 `오버라이딩`이다.

## 오버로딩

- __한 클래스에 이름이 동일한 메소드가 중복되어 정의된 경우__를 `메소드 오버로딩`(overloading)이라 한다. __이 메소드들은 이름이 동일하지만 매개변수 형이나 개수를 다르게 설정하여 구별할 수 있다.__   
</br>

- 메소드끼리 구별할 수 있게 해주는 매개변수 형이나 개수 같은 요소를 `시그니처`(signature)라 한다.

```java
void simpleMethod(int n) {...}
void simpleMethod(int n1, double n2) {...}
```


- 그러나 다음과 같이 반환형이 다른 경우 메소드 오버로딩이 성립하지 않는다.

```java
int simpleMethod() {...}
double simpleMethod() {...}
```
</br>

</br>

- 상속 구조에서 메소드 오버로딩은 한 클래스 안의 오버로딩과 사용 형태가 조금 다르다. 한 클래스에서는 시그니처로 구별했지만, 상속 구조에서는 매개변수 형과 개수가 모두 같다. __대신 상위(부모) 클래스가 추상 클래스(abstract class)여야 하고, 추상 클래스 내의 메소드도 추상 메소드(abstract method)여야 한다.__  
</br>

- __하위 클래스들이 상위 클래스의 추상 메소드를 각각 다르게 구현하며, 특정 개체의 해당 메소드는 프로그램의 실행 시간에 호출된다.__ 이러한 의미에서 __다형성을 동적 바인딩(dynamic binding) 또는 늦은 바인딩(late binding)이라고도 한다.__

## 오버라이딩

- `오버라이딩`(overriding)은 상속과 큰 관련이 있다. __상위 클래스에 정의된 메소드를 하위 클래스에서 재정의하여 사용하는 것이다.__  
</br>

- __메소드의 이름, 매개변수, 반환형은 동일하지만 구현된 내용이 다르다.__  
</br>

- 메소드를 오버라이딩하면 상위 클래스의 메소드는 무시되고, 하위 클래스의 메소드가 사용된다.

```java
class Thought {
    public void message() {
        System.out.println("Hello World");
    }
}

class Talk extends Thought {
    @Override
    public void message() {
        System.out.println("JAVA");
    }
}
```

- @Override는 자바 5에서 소개된 어노테이션 중 하나이다. 자바 컴파일러에게 상위 클래스의 메소드를 오버라이딩한다고 알려주는 일종의 메모이다.


## 오버로딩과 오버라이딩의 차이

- 오버로딩은 상속 관계에서 `추상 클래스`와 `추상 메소드`에서만 사용할 수 있다.
- 반면, 오버라이딩은 추상 클래스와 일반 클래스 `모두에서` 사용할 수 있다.


## 마무리
1. `오버로딩`은 같은 메소드에 `매개변수의 형`과 `개수`를 달리하여 `중복 정의`하는 것이다.
2. `오버라이딩`은 상속 관계에서 `상위 클래스`에 정의된 메소드를 `하위 클래스`에서 `재정의`하여 사용하는 것이다.
